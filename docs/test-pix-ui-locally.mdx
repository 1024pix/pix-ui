# Tester localement Pix UI dans un app

## Tester un composant en developpement dans une app

Il est possible de tester l'intégration d'un composant dans une application
Ember sans nécessairement créer une nouvelle release.

Pour cela, dans le repertoire de l'application Ember, installer Pix UI
en utilisant l'adresse du dépôt Github et le nom de la branche.
Par exemple :

```shell
npm install "git://github.com/1024pix/pix-ui#add-modal-component" --save-dev
```

Une fois l'application Ember redémarrée, il sera possible d'accéder aux
composants Pix UI tels qu'ils sont dans la branche en cours de développement.

## Tester des changements sur Pix UI en developpement dans une app

Il est possible de tester des changements de Pix UI dans une application
Ember sans nécessairement créer une nouvelle release.

Pour cela, dans le répertoire de l'application Pix UI, éxecuter la commande:

```shell
npm pack
```
cela va creer une archive `.tgz` de la librairie

Se déplacer dans le répertoire de l'application dans laquelle l'on veut tester la librairie
puis installer la version compilée localement :

```shell
npm install --save-dev "file:../../pix-ui/1024pix-pix-ui-<X.Y.Z>.tgz"
```
on notera la présence de `file:` suivi du chemin vers l'archive.

Puis lancer l'application

```shell
npm start
```

Une fois l'application Ember redémarrée, il sera possible de voir les changements
de Pix UI tels qu'ils sont dans la branche en cours de développement.
